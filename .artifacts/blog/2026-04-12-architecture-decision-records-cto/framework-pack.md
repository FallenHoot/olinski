# Framework Pack: Architecture Decision Records

## ADR Template (6 sections, 1 page)
1. **Title:** "Use Kubernetes for container orchestration"
2. **Status:** Proposed/Accepted/Superseded
3. **Context:** Why are we deciding this now?
4. **Decision:** What are we doing?
5. **Consequences:** What trade-offs?
6. **Alternatives Considered:** What else did we evaluate?

## Common Architectural Decisions to ADR
- Database technology choice (SQL vs NoSQL)
- Deployment model (VMs vs containers vs serverless)
- API architecture (REST vs gRPC)
- Authentication strategy
- Observability tooling

## CTO Checklist
1. Define ADR process (Git repo, review gate)
2. Write 3 initial ADRs on key decisions
3. Share ADR process with teams
4. Review ADRs in architecture forum
5. Update ADRs when decisions change
