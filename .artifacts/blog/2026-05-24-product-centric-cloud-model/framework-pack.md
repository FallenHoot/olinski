# Framework Pack: Product-Centric Transition

## Org Structure Evolution
### Current (Function-centric)
- Database Team
- Network Team
- Security Team
- DevOps Team
- Product A Team (application only)

### Target (Product-centric)
- Product A Team (owns app + database + network + security for A)
- Product B Team (owns app + database + network + security for B)
- Shared Platform Team (provides shared infrastructure, templates, tooling)

## Pilot Program
1. Pick Product A as pilot
2. Assign DevOps engineer to Product A full-time
3. Give team ownership of database, network, security decisions
4. Measure: deployment velocity, MTTR, team satisfaction
5. Scale to other products if successful

## Checklist This Week
1. Identify pilot product team
2. Discuss with team how shared platform should work
3. Define "infrastructure decisions" product team can make
4. Kick off 6-month pilot
