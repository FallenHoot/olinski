# Audience Signals: BCDR for PostgreSQL Flexible Server
## Audience Problem
Teams using Azure Database for PostgreSQL Flexible Server need BCDR strategies. Questions: Are built-in backups sufficient? When do we use read replicas? Geo-replication? Teams often over-invest in replicas (high cost) or under-invest (unacceptable RTO). Unclear RPO/RTO mapping to each option.
## Audience Alignment
- Database architects designing PostgreSQL infrastructure
- Platform teams managing database services
- DevOps teams owning backup/recovery
