## Core thesis

Enterprise teams need a published, practitioner-friendly governance framework for AI agents, covering decision authority, audit, scope control, and compliance. No such framework exists yet.

## Section flow

1. Why this matters: agents are different from software; governance gaps create risk
2. What changed: traditional software governance is insufficient for agents
3. Framework or model: seven-layer governance approach
4. Practical implementation: start small, build audit logging first
5. Risks and trade-offs: governance has skill and discipline costs
6. What to do this week: write an agent charter

## Counter-position to address

"Agents are just code; use existing software governance."

Response: Agents make runtime decisions based on context; static code review and change management cannot capture agent reasoning or autonomy boundaries.
